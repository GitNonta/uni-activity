# Prompt: ระบบรับสมัครงานทั่วไปและ Part-time (นักศึกษา + Admin)

## บทบาทและเป้าหมาย

คุณคือนักพัฒนา Full-stack ที่เชี่ยวชาญการออกแบบระบบจัดการงาน สร้างฟีเจอร์ระบบประกาศรับสมัครงานทั่วไปและ Part-time ที่รองรับ 2 กลุ่มผู้ใช้หลัก ได้แก่ **นักศึกษา (Student)** และ **Admin / ผู้ดำเนินการจัดงาน (Organizer)** โดยมีการแสดงผลแบบ Card และฟังก์ชันครบครัน

---

## โครงสร้างข้อมูล Job Card

แต่ละประกาศงานต้องแสดงข้อมูลครบถ้วนในรูปแบบ Card ดังนี้:

### ข้อมูลหลัก (Primary Info)
| ฟิลด์ | รายละเอียด | ตัวอย่าง |
|---|---|---|
| `title` | หัวข้อพาดหัวงาน | "พนักงานเสิร์ฟงานเลี้ยง" |
| `jobType` | ประเภทงาน | `งานทั่วไป` / `Part-time` |
| `position` | ตำแหน่งงาน | "พนักงานต้อนรับ" |
| `quota` | จำนวนรับสมัคร | 10 คน |
| `workPeriod` | ช่วงเวลางาน | "08:00 – 17:00 น." |
| `compensation` | ค่าตอบแทน | "400 บาท/วัน" |
| `location` | สถานที่ปฏิบัติงาน | "โรงแรม ABC ถนนสุขุมวิท" |
| `startDate` | ระยะเวลาปฏิบัติงานตั้งแต่ | "1 เม.ย. 2568 – 30 เม.ย. 2568" |
| `dresscode` | การแต่งกาย | "ชุดสุภาพ เสื้อสีขาว กางเกงสีดำ" |
| `gender` | เพศที่รับสมัคร | `ชาย` / `หญิง` / `ไม่จำกัด` |
| `note` | * หมายเหตุงาน | "ต้องมีประสบการณ์อย่างน้อย 1 ครั้ง" |
| `description` | รายละเอียดงาน | ข้อความอธิบายเพิ่มเติม |

### ข้อมูลสถานะและสถิติ (Status & Stats)
| ฟิลด์ | รายละเอียด |
|---|---|
| `totalApplicants` | จำนวนผู้สมัครทั้งหมด |
| `confirmedApplicants` | จำนวนที่ได้รับการยืนยัน (Confirm) แล้ว |
| `status` | สถานะประกาศ: `เปิดรับสมัคร` / `ปิดรับสมัคร` / `เสร็จสิ้น` |
| `mapLocation` | พิกัด GPS สำหรับแสดงแผนที่ |

## ฟีเจอร์ระบบ

### 1. Job Card Display
- แสดงผลแบบ Grid Card ที่ responsive บน Desktop / Tablet / Mobile
- มี Badge แสดงประเภทงาน (งานทั่วไป / Part-time) ด้วยสีต่างกัน
- มี Status Badge: 🟢 เปิดรับสมัคร / 🔴 ปิดรับสมัคร / ⚫ เสร็จสิ้น
- แสดง Progress Bar หรือตัวเลข `ยืนยันแล้ว X / Y คน`
- ใช้ Icon ประกอบแต่ละฟิลด์เพื่อให้อ่านง่าย

### 2. แผนที่สถานที่ (Map Integration)
- แสดงแผนที่ภายใน Card (embed map) โดยใช้ Google Maps หรือ Leaflet.js
- มีปุ่ม "นำทาง" เพื่อเปิด Google Maps แบบ full-screen
- แสดง Marker พร้อมชื่อสถานที่บนแผนที่

### 3. ระบบคอมเมนต์ (Comment System)
- ผู้ใช้ทุกคนสามารถแสดงความคิดเห็นได้
- แสดงชื่อผู้ใช้, วันที่, และเนื้อหาคอมเมนต์
- Admin สามารถลบคอมเมนต์ที่ไม่เหมาะสมได้
- รองรับ reply คอมเมนต์

### 4. ระบบสอบถามข้อมูลเพิ่มเติม (Inquiry System)
- ปุ่ม "สอบถามข้อมูล" เปิด modal หรือ chat box
- ส่งคำถามตรงถึง Admin / ผู้จัดงาน
- Admin ตอบกลับได้ภายในระบบ
- แจ้งเตือนผู้ถามเมื่อมีคำตอบ (Notification)

### 5. ระบบสมัครงาน (Application System)
- ปุ่ม "สมัครงาน" สำหรับนักศึกษา
- ระบบตรวจสอบว่าสมัครแล้วหรือยัง
- แสดงสถานะการสมัคร: `รอการพิจารณา` / `ได้รับการยืนยัน` / `ไม่ผ่านการพิจารณา`

## บทบาทผู้ใช้งาน (User Roles)

### 👨‍🎓 นักศึกษา (Student)
- ดูรายการประกาศงานทั้งหมด
- กดสมัครงาน และติดตามสถานะ
- แสดงความคิดเห็น (Comment)
- สอบถามข้อมูลเพิ่มเติมจาก Admin
- ดูแผนที่สถานที่งาน
- บันทึกงานที่สนใจ (Bookmark)

### 🛠️ Admin / ผู้ดำเนินการจัดงาน (Organizer)
- สร้าง / แก้ไข / ลบประกาศงาน
- กรอกข้อมูลครบทุกฟิลด์ใน Job Card
- จัดการผู้สมัคร: Confirm / Reject
- ดูรายชื่อผู้สมัครทั้งหมดพร้อมข้อมูล
- ตอบคำถามจากนักศึกษา
- ลบคอมเมนต์ที่ไม่เหมาะสม
- เปลี่ยนสถานะประกาศงาน (เปิด/ปิด/เสร็จสิ้น)
- Export รายชื่อผู้สมัครเป็น Excel/PDF

### Design Guidelines
- ใช้ Tailwind CSS หรือ Material UI
- Color Scheme: งานทั่วไป → สีน้ำเงิน, Part-time → สีส้ม
- Font: อ่านง่าย, รองรับภาษาไทย
- Mobile-first responsive design
- Dark mode support (optional)

## Tech Stack แนะนำ

```
Frontend:   React.js + TypeScript + Tailwind CSS
Backend:    Node.js (Express) หรือ Next.js API Routes
Database:   PostgreSQL หรือ MongoDB
Map:        Google Maps API หรือ Leaflet.js + OpenStreetMap
Auth:       NextAuth.js หรือ JWT
Real-time:  Socket.io (สำหรับ comment และ notification)
Storage:    AWS S3 หรือ Cloudinary (สำหรับรูปภาพประกอบงาน)
```

## API Endpoints ที่ต้องการ

```
GET    /api/jobs                    → ดูรายการงานทั้งหมด (พร้อม filter/search)
GET    /api/jobs/:id                → ดูรายละเอียดงาน
POST   /api/jobs                    → สร้างประกาศงาน (Admin only)
PUT    /api/jobs/:id                → แก้ไขประกาศงาน (Admin only)
DELETE /api/jobs/:id                → ลบประกาศงาน (Admin only)
PATCH  /api/jobs/:id/status         → เปลี่ยนสถานะงาน (Admin only)

POST   /api/jobs/:id/apply          → สมัครงาน (Student)
GET    /api/jobs/:id/applicants     → ดูรายชื่อผู้สมัคร (Admin only)
PATCH  /api/jobs/:id/applicants/:uid → Confirm/Reject ผู้สมัคร (Admin only)

GET    /api/jobs/:id/comments       → ดูคอมเมนต์
POST   /api/jobs/:id/comments       → เพิ่มคอมเมนต์
DELETE /api/jobs/:id/comments/:cid  → ลบคอมเมนต์ (Admin / เจ้าของ)

POST   /api/jobs/:id/inquiries      → ส่งคำถาม (Student)
POST   /api/jobs/:id/inquiries/:iid/reply → ตอบคำถาม (Admin)
```

## Filter & Search ที่ควรมี

- ค้นหาด้วยชื่องาน / สถานที่
- กรองตามประเภทงาน (ทั่วไป / Part-time)
- กรองตามช่วงเวลา / วันที่
- กรองตามค่าตอบแทน (min-max)
- กรองตามเพศที่รับ
- กรองตามสถานะ (เปิด/ปิด)
- เรียงลำดับตามวันที่ประกาศ / ค่าตอบแทน

## Notification System

- แจ้งเตือนนักศึกษาเมื่อ: Admin ยืนยัน / ปฏิเสธการสมัคร
- แจ้งเตือนนักศึกษาเมื่อ: Admin ตอบคำถาม
- แจ้งเตือน Admin เมื่อ: มีนักศึกษาสมัครงานใหม่
- แจ้งเตือน Admin เมื่อ: มีคำถามใหม่เข้ามา

## หมายเหตุสำคัญ

> สร้างโค้ดที่ clean, มี comment อธิบาย, และแยก component ชัดเจน
> ทุกฟิลด์ที่มี * คือ optional field ที่ Admin อาจไม่กรอกก็ได้
> ระบบต้องรองรับภาษาไทยทั้งหมด ทั้ง input และ display
> ต้องมี validation ป้องกัน quota เกินจำนวนที่กำหนด
